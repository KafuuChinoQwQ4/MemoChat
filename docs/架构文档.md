## 一、项目概览

MemoChat 是一个**企业级即时通讯（IM）平台**，采用微服务架构，同时支持 HTTP/1.1、HTTP/3（QUIC）、TCP 和 gRPC 多协议客户端接入。项目分为客户端（Qt/QML 桌面应用）和服务端（C++ 微服务集群）两大部分，具备完整的可观测性、容器化部署和压测工具链。

### 4.2 传输层架构

客户端同时支持三种传输通道，通过 `TransportSelector` 管理主备切换：

```
┌───────────────────────────────────────────────────────────────┐
│                    TransportSelector                            │
│                                                               │
│  ┌──────────────────────┐  ┌──────────────────────────────┐ │
│  │   TcpMgr             │  │  QuicChatTransport           │ │
│  │   (QTcpSocket)       │  │  (MsQuic)                    │ │
│  │   - Binary frame     │  │  - Binary frame              │ │
│  │   - 4B header        │  │  - ALPN: memochat-chat      │ │
│  │   - fallback         │  │  - primary (preferred)       │ │
│  └──────────────────────┘  └──────────────────────────────┘ │
│  ┌──────────────────────┐                                    │
│  │   HttpMgr            │                                    │
│  │   (QNetworkAccessMgr)│                                    │
│  │   - JSON over HTTP   │                                    │
│  │   - 注册/登录/验证码  │                                    │
│  └──────────────────────┘                                    │
└───────────────────────────────────────────────────────────────┘
```

**二进制帧格式**（TCP/QUIC 共用）：

```
| MSG_ID (2B big-endian) | DATA_LEN (2B big-endian) | PAYLOAD (JSON) |
```

### 6.4 TCP/QUIC 二进制协议

**帧格式**（定义于 `server/ChatServer/const.h`）：

```
+--------+--------+--------+--------+============+
| MSG_ID |  DATA_LEN (big-endian)   |  PAYLOAD   |
| 2 bytes|    2 bytes              |  N bytes   |
+--------+-------------------------+============+
```

- `HEAD_TOTAL_LEN = 4`，`HEAD_ID_LEN = 2`，`HEAD_DATA_LEN = 2`
- `MAX_LENGTH = 30 * 1024`

**TCP 消息 ID 枚举**（`ReqId` / `MSG_IDS`）：

| ID | 名称 | 方向 | 功能 |
|----|------|------|------|
| 1001 | ID_GET_VARIFY_CODE | C→S | 获取验证码 |
| 1002 | ID_REG_USER | C→S | 注册 |
| 1003 | ID_RESET_PWD | C→S | 重置密码 |
| 1004 | ID_LOGIN_USER | C→Gate | HTTP 登录 |
| **1005** | **ID_CHAT_LOGIN** | **C→Chat** | **ChatServer 登录** |
| **1006** | **ID_CHAT_LOGIN_RSP** | **Chat→C** | **登录响应** |
| 1007/1008 | ID_SEARCH_USER | 搜索用户 |
| 1009/1010 | ID_ADD_FRIEND | 添加好友 |
| 1011 | ID_NOTIFY_ADD_FRIEND_REQ | 好友申请通知 |
| 1013/1014 | ID_AUTH_FRIEND | 认证好友 |
| 1015 | ID_NOTIFY_AUTH_FRIEND_REQ | 好友认证通知 |
| **1017/1018** | **ID_TEXT_CHAT_MSG** | **私聊消息** |
| **1019** | **ID_NOTIFY_TEXT_CHAT_MSG_REQ** | **私聊消息通知** |
| **1023/1024** | **ID_HEART_BEAT** | **心跳保活** |
| **1044/1045** | **ID_GROUP_CHAT_MSG** | **群聊消息** |
| **1046** | **ID_NOTIFY_GROUP_CHAT_MSG_REQ** | **群聊消息通知** |
| 1031/1032 | ID_CREATE_GROUP | 建群 |
| 1033/1034 | ID_GET_GROUP_LIST | 群列表 |
| 1047/1048 | ID_GROUP_HISTORY | 群历史 |
| 1059/1060 | ID_PRIVATE_HISTORY | 私聊历史 |
| 1063/1064 | ID_GET_DIALOG_LIST | 对话列表 |
| 1092/1093 | ID_GET_RELATION_BOOTSTRAP | 关系引导数据 |
| 1094 | ID_NOTIFY_MSG_STATUS_REQ | 消息状态通知 |

### 6.5 ChatLoginTicket（会话票据）

采用类 JWT 的 HMAC-SHA256 签名机制（`server/common/auth/ChatLoginTicket.h`）：

```
<payload_base64url>.<<HMAC-SHA256(secret, payload)>_base64url>
```

- Windows: BCrypt；Linux: OpenSSL
- Base64URL 编码（无需填充）

**Protocol v3 支持**：签名验证、过期时间验证、目标服务器名称验证（防重放）。

## 十七、消息流程详解

### 17.1 用户登录流程

1. 客户端发送 HTTP POST /user_login 到 GateServer
2. GateServer 调用 VarifyServer 验证验证码
3. GateServer 调用 StatusServer 获取 ChatServer 路由
4. GateServer 返回登录票据和 ChatServer 信息
5. 客户端通过 TCP/QUIC 连接 ChatServer 并验证票据
6. ChatServer 记录用户在线状态到 Redis

### 17.2 私聊消息流程

1. 发送方通过 TCP/QUIC 发送消息到 ChatServer
2. ChatServer 发布消息到 Kafka (chat.private.v1)
3. ChatServer 消费消息并持久化到 PostgreSQL 和 MongoDB
4. ChatServer 通过 TCP/QUIC 通知接收方
5. 接收方发送已读回执
6. ChatServer 通过 RabbitMQ 处理离线通知

### 17.3 群聊消息流程

1. 发送方发送群消息到 ChatServer
2. ChatServer 发布消息到 Kafka (chat.group.v1)
3. ChatServer 消费消息并通知所有群成员
4. 离线成员通过 RabbitMQ 收到通知

## 十九、安全机制

### 19.1 认证流程

1. 用户认证: HTTP 登录 -> 验证码验证 -> 返回 login_ticket
2. 会话认证: TCP/QUIC 登录 -> 验证 login_ticket -> 建立会话
3. gRPC 认证: 服务间调用 -> mTLS 或预共享密钥

### 19.2 授权模型

- 用户只能访问自己的聊天记录
- 群成员只能访问群内消息
- 管理员可以访问系统管理接口

### 19.3 数据安全

- 传输加密: TLS 1.3 / QUIC
- 存储加密: 敏感数据 AES-256 加密
- 日志脱敏: 手机号、身份证号、密码等敏感信息自动脱敏
