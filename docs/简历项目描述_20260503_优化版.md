# MemoChat 项目简历精简版（两项目一页）

## 项目一：分布式全栈 IM 系统

**项目时间**：2025.11 - 2026.05  
**项目角色**：核心后端开发 / 全栈开发  
**技术栈**：C++23、Qt/QML、Boost.Asio、gRPC、PostgreSQL、MongoDB、Redis、MinIO、Redpanda/Kafka、RabbitMQ、Prometheus、Grafana、Loki、Tempo、Python

- 负责自研桌面端分布式 IM 系统，支持注册登录、验证码、好友关系、私聊、群聊、离线消息、历史消息、媒体上传下载等 `10+` 核心模块；将本地集群扩展为 `2 GateServer + 6 ChatServer + 2 StatusServer + 2 VarifyServer`，ChatServer 覆盖 TCP `8090-8094/8097`、QUIC `8190-8195`、gRPC `50055-50059/50581`。
- 设计 GateServer 接入层与 StatusServer 路由链路，通过短效 `login_ticket` 将 ChatServer 登录鉴权从 Redis/MySQL 重复校验优化为本地验票；HTTP 登录在单机 Windows + Docker 环境下达到 `1240.895 QPS`，`P50=34.070ms`，`P95=79.059ms`，成功率 `100%`。
- 优化登录审计与消息总线链路，将 Kafka 审计从同步发送改为异步投递，移除热路径 `flush()`，结合双 Gate 轮询、keep-alive 压测器和连接池调优，使高并发 HTTP 登录保持 `1082.389 QPS`，`P95=306.280ms`。
- 优化 ChatServer TCP 登录主路径，删除重复 Redis 路由写入、登录后验证读、高频日志，并将离线补推与唯一用户压测下的互踢检查从主路径解耦；50 并发 TCP 登录从 `43.362 QPS / P95=1508.947ms` 提升到 `371.054 QPS / P95=177.372ms`，吞吐提升约 `755.6%`，P95 降低约 `88.2%`。
- 建设 Docker 化数据与消息基础设施，使用 PostgreSQL 存用户/关系/索引，MongoDB 存消息正文，Redis 存在线状态/路由/缓存，MinIO 存媒体文件，Redpanda/RabbitMQ 承载事实流与异步任务；本地基准中 Redis GET/SET 达 `2.1k-2.3k QPS`，MongoDB insert/findOne 达 `1.45k-2.06k QPS`。
- 搭建 Python-only 压测与 Prometheus/Grafana/Loki/Tempo 可观测链路，覆盖 HTTP/TCP/AI/RAG/存储基准；通过 trace 与结构化日志定位并修复 Windows 保留端口、ChatServer `1072/1074` variant 崩溃、控制台 I/O 阻塞等问题，恢复 `6` 个 ChatServer 全量监听与 `100%` 登录成功率。

## 项目二：IM 场景 AI Agent 与 RAG 协同平台

**项目时间**：2026.03 - 2026.05  
**项目角色**：AI Agent 后端开发 / 全栈开发  
**技术栈**：FastAPI、Uvicorn、Python、Ollama、Qdrant、Neo4j、PostgreSQL、OpenTelemetry、Prometheus、Qt/QML

- 在 MemoChat IM 基础上建设 AI Agent 与 RAG 协同平台，提供智能问答、知识库检索、图谱记忆、工具调用、聊天摘要、智能回复、翻译等 `7` 类 AI 场景；AIOrchestrator 提供 `/chat`、`/smart`、`/kb`、`/models`、`/recommend`、`/agent` 等 `6` 组核心路由。
- 设计 Agent Harness 分层架构，将 orchestration、execution、feedback、memory、knowledge、llm、mcp、skills、runtime 拆成 `9` 个模块，统一封装 Ollama、OpenAI、Claude、Kimi 等模型 provider，支持模型不可用时通过 `/ready` 健康检查隔离 AI 故障。
- 实现 RAG 知识库链路，完成文档上传、解析、切片、Embedding、Qdrant 写入和 TopK 检索；固定知识集压测中 `/kb/search` 达到 `24.020 QPS`，`P50=350.436ms`，`P95=596.849ms`，`recall@k=100%`，`hallucination_rate=0%`。
- 建设“向量召回 + 图谱记忆”双检索能力，使用 Qdrant 处理语义相似度检索、Neo4j 保存用户偏好/实体关系/长期上下文；本地 Qdrant Top5 向量搜索达到 `824.782 QPS`，`P95=36.043ms`。
- 设计 Agent trace 与 feedback 闭环，落库 `ai_agent_run_trace`、`ai_agent_run_step`、`ai_agent_feedback` 等执行数据，支持按 `trace_id` 回放计划生成、工具调用、模型响应和用户评分；AIOrchestrator `/health` 压测达到 `260.516 QPS`，`P95=135.959ms`，可用性 `100%`。
- 打通 Qt/QML 客户端 AI 入口，完成 Agent 对话、知识库管理、轨迹查看、记忆管理等 UI 模块，与 Gate/AIServer/AIOrchestrator 解耦联动，使 IM 客户端从单一聊天工具扩展为“聊天 + AI 助手 + 知识库 + 工具执行”的复合型应用。
