# ADR 0001: 开发阶段不向后兼容旧协议

## 状态

已采用。

## 决策

MemoChat 当前按 3.0 开发节奏推进，不向后兼容 2.0 或更早客户端。新增或修改协议字段时，直接使用最新字段名和数据格式，不写“读新回退旧”的双读逻辑，不保留旧字段别名，也不为历史数据格式新增迁移读路径。

## 原因

- 项目仍处于快速演进和模块拆分阶段，保留旧协议兼容会显著增加代码路径和测试矩阵。
- 旧客户端通过强制更新机制拦截，比在业务代码里长期保留兼容分支更清晰。
- 服务端、QML 客户端和 Web 前端都应围绕最新契约收敛，避免同一字段存在多个事实来源。

## 例外

以下不是旧协议兼容，不能误删：

- 平台适配，例如 Linux/WSLg、Windows、macOS 的 UI/图形差异。
- 第三方库或系统 API 的兼容包装。
- 错误兜底、超时、重试和强制更新机制本身。
- 本地开发环境和生产环境之间的配置注入差异。
