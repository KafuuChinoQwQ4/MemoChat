# MemoChat 数据库架构文档

> 文档版本：2026-03-23

---

## 一、数据库概览

MemoChat 采用多数据库架构，针对不同数据特性选择最适合的存储引擎：

| 数据库 | 引擎 | 用途 | 数据特点 |
|--------|------|------|----------|
| PostgreSQL | PostgreSQL 16 | 用户、关系、群组、会话 | 结构化、强一致性 |
| MongoDB | MongoDB 7 | 聊天消息历史 | 半结构化、高写入 |
| Redis | Redis 7 | 缓存、会话、实时状态 | 内存型、低延迟 |

### 1.1 PostgreSQL Schema

- **Schema 名**: `memo` (可通过配置修改)
- **用途**: 业务主数据存储

### 1.2 MongoDB Collections

- **用途**: 聊天消息历史存储，支持海量消息查询

### 1.3 Redis Keys

- **用途**: 缓存、Session、实时状态、发布订阅

## 二、PostgreSQL 表结构

### 2.1 用户表 (user)

存储用户基本信息。

```sql
CREATE TABLE IF NOT EXISTS "user" (
    id integer GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    uid integer NOT NULL DEFAULT 0,           -- 用户唯一ID
    name varchar(255) NOT NULL DEFAULT '',    -- 用户名
    email varchar(255) NOT NULL DEFAULT '',    -- 邮箱
    pwd varchar(255) NOT NULL DEFAULT '',     -- 密码哈希
    nick varchar(255) NOT NULL DEFAULT '',    -- 昵称
    "desc" varchar(255) NOT NULL DEFAULT '',  -- 个人描述
    sex integer NOT NULL DEFAULT 0,            -- 性别 0=未知 1=男 2=女
    icon varchar(255) NOT NULL DEFAULT '',     -- 头像URL
    user_id varchar(10),                       -- 用户名ID (唯一)
    CONSTRAINT uq_user_uid UNIQUE (uid),
    CONSTRAINT uq_user_email UNIQUE (email),
    CONSTRAINT uq_user_user_id UNIQUE (user_id)
);
```

**索引**:
- `idx_user_name` - 按用户名搜索

**字段说明**:
| 字段 | 类型 | 说明 |
|------|------|------|
| uid | integer | 全局唯一用户ID |
| name | varchar | 用户名（登录名） |
| email | varchar | 邮箱地址 |
| pwd | varchar | BCrypt 哈希密码 |
| nick | varchar | 显示昵称 |
| desc | varchar | 个人签名 |
| sex | integer | 性别 0=未知 1=男 2=女 |
| icon | varchar | 头像URL |
| user_id | varchar | 短用户名ID |

### 2.2 UID 生成器 (user_id)

用于生成全局唯一的用户ID。

```sql
CREATE TABLE IF NOT EXISTS "user_id" (
    id integer PRIMARY KEY
);
```

### 2.3 好友关系表 (friend)

存储双向好友关系。

```sql
CREATE TABLE IF NOT EXISTS friend (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    self_id integer NOT NULL,      -- 自己的uid
    friend_id integer NOT NULL,    -- 好友的uid
    back varchar(255) NOT NULL DEFAULT '',  -- 备注名
    CONSTRAINT uq_friend_self_friend UNIQUE (self_id, friend_id)
);
```

**索引**:
- `idx_friend_self_id_friend_id` - 复合索引加速查询

**字段说明**:
| 字段 | 类型 | 说明 |
|------|------|------|
| self_id | integer | 用户自己的uid |
| friend_id | integer | 好友的uid |
| back | varchar | 备注名 |

### 2.4 好友申请表 (friend_apply)

存储好友申请记录。

```sql
CREATE TABLE IF NOT EXISTS friend_apply (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    from_uid integer NOT NULL,      -- 申请方uid
    to_uid integer NOT NULL,        -- 被申请方uid
    status smallint NOT NULL DEFAULT 0,  -- 状态 0=待处理 1=已同意 2=已拒绝
    CONSTRAINT uq_friend_apply_from_to UNIQUE (from_uid, to_uid)
);
```

**索引**:
- `idx_friend_apply_to_uid_id_from_uid` - 加速查询待处理申请

**状态说明**:
| 状态值 | 说明 |
|--------|------|
| 0 | 待处理 |
| 1 | 已同意 |
| 2 | 已拒绝 |

### 2.5 好友申请标签表 (friend_apply_tag)

存储好友申请的附加标签。

```sql
CREATE TABLE IF NOT EXISTS friend_apply_tag (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    from_uid integer NOT NULL,
    to_uid integer NOT NULL,
    tag varchar(64) NOT NULL,
    CONSTRAINT uq_friend_apply_tag UNIQUE (from_uid, to_uid, tag)
);
```

### 2.6 好友标签表 (friend_tag)

存储用户对好友的分类标签。

```sql
CREATE TABLE IF NOT EXISTS friend_tag (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    self_id integer NOT NULL,
    friend_id integer NOT NULL,
    tag varchar(64) NOT NULL,
    CONSTRAINT uq_friend_tag UNIQUE (self_id, friend_id, tag)
);
```

### 2.7 群组表 (chat_group)

存储群组基本信息。

```sql
CREATE TABLE IF NOT EXISTS chat_group (
    group_id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    group_code varchar(10),                     -- 群号
    name varchar(64) NOT NULL,                  -- 群名称
    owner_uid integer NOT NULL,                  -- 群主uid
    announcement text,                           -- 群公告
    member_limit integer NOT NULL DEFAULT 200,  -- 成员上限
    is_all_muted boolean NOT NULL DEFAULT false, -- 全员禁言
    status smallint NOT NULL DEFAULT 1,         -- 状态 1=正常 0=解散
    created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    icon varchar(512),                          -- 群头像
    CONSTRAINT uq_chat_group_code UNIQUE (group_code)
);
```

**索引**:
- `idx_chat_group_owner_uid` - 按群主查询
- `idx_chat_group_status` - 按状态查询

### 2.8 群组成员表 (chat_group_member)

存储群组成员信息。

```sql
CREATE TABLE IF NOT EXISTS chat_group_member (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    group_id bigint NOT NULL,
    uid integer NOT NULL,
    role smallint NOT NULL DEFAULT 1,       -- 角色 1=普通成员 2=管理员 3=群主
    mute_until bigint NOT NULL DEFAULT 0,    -- 禁言截止时间戳
    join_source smallint NOT NULL DEFAULT 0, -- 入群方式 0=直接加入 1=邀请
    status smallint NOT NULL DEFAULT 1,      -- 状态 1=正常 0=已退出
    joined_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_chat_group_member UNIQUE (group_id, uid)
);
```

**索引**:
- `idx_chat_group_member_uid_status` - 按用户查询其群组
- `idx_chat_group_member_group_status` - 按群组查询成员

**角色说明**:
| 角色值 | 说明 |
|--------|------|
| 1 | 普通成员 |
| 2 | 管理员 |
| 3 | 群主 |

### 2.9 入群申请表 (chat_group_apply)

存储入群申请记录。

```sql
CREATE TABLE IF NOT EXISTS chat_group_apply (
    apply_id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    group_id bigint NOT NULL,
    applicant_uid integer NOT NULL,      -- 申请人uid
    inviter_uid integer NOT NULL DEFAULT 0, -- 邀请人uid
    type smallint NOT NULL DEFAULT 1,   -- 类型 1=申请入群 2=邀请入群
    status smallint NOT NULL DEFAULT 0, -- 状态 0=待处理 1=已同意 2=已拒绝
    reason varchar(128) NOT NULL DEFAULT '', -- 申请理由
    reviewer_uid integer NOT NULL DEFAULT 0, -- 审核人uid
    created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

**索引**:
- `idx_chat_group_apply_group_status` - 按群组和状态查询申请
- `idx_chat_group_apply_applicant` - 按申请人查询
- `idx_chat_group_apply_inviter` - 按邀请人查询

### 2.10 群消息表 (chat_group_msg)

存储群聊消息（仅索引数据，详细内容在 MongoDB）。

```sql
CREATE TABLE IF NOT EXISTS chat_group_msg (
    server_msg_id bigint GENERATED BY DEFAULT AS IDENTITY,
    msg_id varchar(64) PRIMARY KEY,         -- 消息ID
    group_id bigint NOT NULL,                -- 群ID
    from_uid integer NOT NULL,               -- 发送者uid
    msg_type varchar(16) NOT NULL,           -- 消息类型
    content text NOT NULL,                   -- 消息内容
    mentions_json jsonb,                      -- @提及用户列表
    created_at bigint NOT NULL,              -- 创建时间戳
    group_seq bigint NOT NULL DEFAULT 0,     -- 群内序号
    reply_to_server_msg_id bigint NOT NULL DEFAULT 0, -- 回复的消息ID
    forward_meta_json jsonb,                 -- 转发元数据
    edited_at_ms bigint NOT NULL DEFAULT 0,  -- 编辑时间
    deleted_at_ms bigint NOT NULL DEFAULT 0, -- 删除时间
    CONSTRAINT uq_chat_group_msg_server_msg_id UNIQUE (server_msg_id),
    CONSTRAINT uq_chat_group_msg_group_seq UNIQUE (group_id, group_seq)
);
```

**索引**:
- `idx_chat_group_msg_group_created` - 按群组和时间查询
- `idx_chat_group_msg_group_seq` - 按群组序号查询

### 2.11 群消息扩展表 (chat_group_msg_ext)

存储群消息的扩展信息（如文件名、大小等）。

```sql
CREATE TABLE IF NOT EXISTS chat_group_msg_ext (

    file_name varchar(255) NOT NULL DEFAULT '',

```

### 2.12 私聊消息表 (chat_private_msg)

存储私聊消息索引（详细内容在 MongoDB）。

```sql
CREATE TABLE IF NOT EXISTS chat_private_msg (
    msg_id varchar(64) PRIMARY KEY,
    conv_uid_min integer NOT NULL,           -- 对话用户ID小者
    conv_uid_max integer NOT NULL,           -- 对话用户ID大者
    from_uid integer NOT NULL,               -- 发送者uid
    to_uid integer NOT NULL,                 -- 接收者uid
    content text NOT NULL,                   -- 消息内容
    created_at bigint NOT NULL,              -- 创建时间戳
    reply_to_server_msg_id bigint NOT NULL DEFAULT 0,
    forward_meta_json jsonb,                  -- 转发元数据
    edited_at_ms bigint NOT NULL DEFAULT 0,
    deleted_at_ms bigint NOT NULL DEFAULT 0
);
```

**索引**:
- `idx_chat_private_msg_conv_created` - 按对话和时间查询
- `idx_chat_private_msg_to_created` - 按接收者和时间查询

### 2.13 对话列表表 (chat_dialog)

存储用户的对话列表。

```sql
CREATE TABLE IF NOT EXISTS chat_dialog (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    owner_uid integer NOT NULL,              -- 所有者uid
    dialog_type varchar(16) NOT NULL,        -- 对话类型 private/group
    peer_uid integer NOT NULL DEFAULT 0,      -- 私聊对方uid
    group_id bigint NOT NULL DEFAULT 0,       -- 群ID
    pinned_rank integer NOT NULL DEFAULT 0,   -- 置顶排序
    draft_text varchar(2000) NOT NULL DEFAULT '', -- 草稿
    mute_state smallint NOT NULL DEFAULT 0,  -- 静音状态
    last_msg_preview varchar(255) NOT NULL DEFAULT '', -- 最后消息预览
    last_msg_ts bigint NOT NULL DEFAULT 0,   -- 最后消息时间
    unread_count integer NOT NULL DEFAULT 0, -- 未读数
    updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_chat_dialog_owner UNIQUE (owner_uid, dialog_type, peer_uid, group_id)
);
```

**索引**:
- `idx_chat_dialog_owner` - 按用户查询
- `idx_chat_dialog_owner_type` - 按用户和类型查询

### 2.14 对话元数据表 (chat_dialog_meta)

存储对话的元数据。

```sql
);
```

### 2.18 媒体资产表 (chat_media_asset)

存储媒体文件元数据。

```sql
CREATE TABLE IF NOT EXISTS chat_media_asset (
    media_id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    media_key varchar(64) NOT NULL,          -- 媒体唯一键
    owner_uid integer NOT NULL,               -- 所有者uid
    media_type varchar(16) NOT NULL,          -- 媒体类型 image/video/file
    origin_file_name varchar(255) NOT NULL,   -- 原始文件名
    mime varchar(127) NOT NULL DEFAULT '',
    size_bytes bigint NOT NULL DEFAULT 0,
    storage_provider varchar(32) NOT NULL DEFAULT 'local', -- 存储提供者
    storage_path varchar(1024) NOT NULL,      -- 存储路径
    created_at_ms bigint NOT NULL,
    deleted_at_ms bigint NOT NULL DEFAULT 0,
    status smallint NOT NULL DEFAULT 1,       -- 状态 1=正常 0=已删除

```

**索引**:
- `idx_chat_media_asset_owner_created` - 按所有者和时间查询

### 2.19 通话会话表 (chat_call_session)

存储音视频通话记录。

```sql
CREATE TABLE IF NOT EXISTS chat_call_session (
    call_id varchar(64) PRIMARY KEY,
    room_name varchar(128) NOT NULL,         -- LiveKit 房间名
    call_type varchar(16) NOT NULL,          -- 通话类型 audio/video
    caller_uid integer NOT NULL,              -- 主叫uid
    callee_uid integer NOT NULL,              -- 被叫uid
    state varchar(32) NOT NULL,               -- 状态 pending/accepted/ended
    started_at_ms bigint NOT NULL,            -- 开始时间
    accepted_at_ms bigint NOT NULL DEFAULT 0, -- 接听时间
    ended_at_ms bigint NOT NULL DEFAULT 0,   -- 结束时间
    expires_at_ms bigint NOT NULL DEFAULT 0, -- 过期时间
    duration_sec integer NOT NULL DEFAULT 0,  -- 通话时长
    reason varchar(64) NOT NULL DEFAULT '',  -- 结束原因
    trace_id varchar(64) NOT NULL DEFAULT '', -- 链路追踪ID
    updated_at_ms bigint NOT NULL DEFAULT 0
);
```

**索引**:
- `idx_chat_call_session_caller` - 按主叫查询
- `idx_chat_call_session_callee` - 按被叫查询
- `idx_chat_call_session_state` - 按状态查询

**状态说明**:
| 状态值 | 说明 |
|--------|------|
| pending | 等待接听 |
| accepted | 已接听 |
| ended | 已结束 |
| cancelled | 已取消 |
| rejected | 已拒绝 |

### 2.20 聊天事件发件箱表 (chat_event_outbox)

用于 Kafka 事件可靠投递的发件箱模式。

```sql
CREATE TABLE IF NOT EXISTS chat_event_outbox (
    event_id varchar(64) PRIMARY KEY,
    event_type varchar(32) NOT NULL,          -- 事件类型
    topic varchar(64) NOT NULL,               -- Kafka Topic
    payload jsonb NOT NULL,                   -- 事件载荷
    status smallint NOT NULL DEFAULT 0,        -- 状态 0=待发送 1=已发送 2=失败
    retry_count integer NOT NULL DEFAULT 0,    -- 重试次数
    created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
    sent_at timestamptz
);
```

## 四、Redis Keys

### 4.1 会话缓存

| Key Pattern | 类型 | 说明 | TTL |
|-------------|------|------|-----|
| `session:{uid}:{device_id}` | Hash | 用户会话信息 | 24h |
| `login_ticket:{ticket}` | String | 登录票据验证 | 20s |
| `verify_code:{email}` | String | 验证码 | 300s |

### 4.2 在线状态

| Key Pattern | 类型 | 说明 | TTL |
|-------------|------|------|-----|
| `online:{uid}` | String | 用户在线状态 | 60s |
| `chatserver:{uid}` | String | 用户所在 ChatServer | 60s |

### 4.3 关系缓存

| Key Pattern | 类型 | 说明 | TTL |
|-------------|------|------|-----|
| `friend:{uid}` | Set | 用户好友列表 | 1h |
| `group:{group_id}` | Set | 群组成员列表 | 1h |

### 4.4 消息缓存

| Key Pattern | 类型 | 说明 | TTL |
|-------------|------|------|-----|
| `dialog:{uid}` | Sorted Set | 用户对话列表 | 1h |
| `msg:{msg_id}` | Hash | 消息详情缓存 | 5min |

### 4.5 发布订阅通道

| Channel | 说明 |
|---------|------|
| `chat:notify:{uid}` | 用户消息通知 |
| `group:notify:{group_id}` | 群组消息通知 |
| `presence:{uid}` | 用户在线状态变更 |

## 五、索引优化

### 5.1 PostgreSQL 索引策略

| 表名 | 索引类型 | 索引列 | 用途 |
|------|---------|--------|------|
| user | UNIQUE | email | 邮箱唯一约束 |
| user | UNIQUE | uid | UID 唯一约束 |
| user | UNIQUE | user_id | 用户名唯一约束 |
| friend | UNIQUE | (self_id, friend_id) | 好友关系唯一约束 |
| friend | INDEX | (self_id, friend_id) | 好友查询 |
| chat_group_member | UNIQUE | (group_id, uid) | 群成员唯一约束 |
| chat_group_member | INDEX | (uid, status) | 用户所在群查询 |
| chat_private_msg | INDEX | (conv_uid_min, conv_uid_max, created_at) | 私聊历史查询 |
| chat_group_msg | INDEX | (group_id, group_seq) | 群消息序号查询 |

### 5.2 MongoDB 索引策略

| Collection | 索引类型 | 索引列 | 用途 |
|------------|---------|--------|------|
| private_messages | INDEX | (from_uid, to_uid, server_time) | 私聊历史 |
| private_messages | INDEX | (msg_id) | 消息详情查询 |
| group_messages | INDEX | (group_id, server_time) | 群聊历史 |
| group_messages | INDEX | (msg_id) | 消息详情查询 |

## 七、备份与恢复

### 7.1 PostgreSQL 备份

```bash
# 全量备份
pg_dump -h 127.0.0.1 -U memochat -d memo_pg -F c -f backup.sql

# 增量备份（基于 WAL）
pg_basebackup -h 127.0.0.1 -U repl -D /backup -P -X stream
```

### 7.2 MongoDB 备份

```bash
# 全量备份
mongodump --host 127.0.0.1 --port 27017 --db memo --out /backup

# 恢复
mongorestore --host 127.0.0.1 --port 27017 /backup/memo
```

### 7.3 Redis 备份
