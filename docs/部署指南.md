## 一、部署架构概览

MemoChat 支持多种部署方式，从开发环境到生产环境：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           部署模式                                       │
├─────────────────────┬─────────────────────┬─────────────────────────────┤
│   本地开发          │   Docker Compose    │   Kubernetes               │
│   (Windows)         │   (多容器)          │   (生产级)                 │
├─────────────────────┼─────────────────────┼─────────────────────────────┤
│ 单机运行            │ 容器化服务          │ 容器编排                   │
│ 快速迭代            │ 一键启动            │ 自动扩缩容                 │
│ 调试友好            │ 跨平台              │ 服务发现                   │
│ 资源占用小          │ 隔离性好            │ 高可用                     │
└─────────────────────┴─────────────────────┴─────────────────────────────┘
```

## 二、本地开发环境部署

### 2.1 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| 操作系统 | Windows 10/11 | 推荐 Windows 11 |
| CPU | 4 核以上 | 支持多线程编译 |
| 内存 | 16GB 以上 | 服务运行需要 |
| 磁盘 | 50GB 以上 | 代码+构建+运行时 |
| Visual Studio | 2022 | MSVC 编译器 |
| CMake | 3.16+ | 构建系统 |
| vcpkg | 最新稳定版 | C++ 包管理 |
| Qt | 6.8+ | Qt6 框架 |
| Python | 3.9+ | 运维脚本 |

### 2.2 基础设施启动

#### 2.2.1 使用 Docker Compose 启动基础设施

```powershell
# 启动数据存储
docker compose -f .\deploy\local\compose\datastores.yml up -d

# 启动消息队列
docker compose -f .\deploy\local\compose\kafka.yml up -d
docker compose -f .\deploy\local\compose\rabbitmq.yml up -d

# 启动可观测性组件（可选）
docker compose -f .\deploy\local\compose\observability.yml up -d
```

#### 2.2.2 基础设施端口映射

| 服务 | 端口 | 说明 |
|------|------|------|
| PostgreSQL | 5432 | 主数据库 |
| Redis | 6379 | 缓存/会话 |
| MongoDB | 27017 | 消息历史 |
| Redpanda (Kafka) | 19092 | 消息队列 |
| RabbitMQ | 5672, 15672 | 任务队列 |
| Prometheus | 9090 | 指标采集 |
| Grafana | 3000 | 可视化 |
| Loki | 3100 | 日志聚合 |
| Tempo | 3200 | 链路追踪 |

### 2.3 数据库初始化

```powershell
# 初始化 PostgreSQL Schema
powershell -ExecutionPolicy Bypass -File scripts\init_postgresql_schema.ps1

# 初始化 Kafka Topics
powershell -ExecutionPolicy Bypass -File scripts\init_kafka_topics.ps1

# 初始化 RabbitMQ Queues
powershell -ExecutionPolicy Bypass -File scripts\init_rabbitmq_topology.ps1
```

### 2.4 服务构建

#### 2.4.1 CMake 构建

```powershell
# 配置构建
cmake --preset msvc2022-all

# 构建所有目标
cmake --build --preset msvc2022-all-release

# 或者分步构建
cmake -B build -S . -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

#### 2.4.2 构建产物

| 服务 | 可执行文件 | 位置 |
|------|-----------|------|
| GateServer | GateServer.exe | build/bin/Release/ |
| StatusServer | StatusServer.exe | build/bin/Release/ |
| ChatServer | ChatServer.exe | build/bin/Release/ |
| VarifyServer | Node.js 服务 | server/VarifyServer/ |
| MemoChatQml | MemoChatQml.exe | build/bin/Release/ |

### 2.5 服务启动

#### 2.5.1 一键启动所有服务

```powershell
# 启动后端服务
scripts\windows\start_test_services.bat

# 启动完整服务（后端 + 前端）
scripts\windows\start_all_services.bat
```

#### 2.5.2 单服务启动

```powershell
# GateServer
.\build\bin\Release\GateServer.exe --config server\GateServer\config.ini

# StatusServer
.\build\bin\Release\StatusServer.exe --config server\StatusServer\config.ini

# ChatServer (4 个节点)
.\build\bin\Release\ChatServer.exe --config server\ChatServer\chatserver1.ini
.\build\bin\Release\ChatServer.exe --config server\ChatServer\chatserver2.ini
.\build\bin\Release\ChatServer.exe --config server\ChatServer\chatserver3.ini
.\build\bin\Release\ChatServer.exe --config server\ChatServer\chatserver4.ini

# VarifyServer (Node.js)
cd server\VarifyServer
npm install
node server.js

# MemoChatQml 客户端
.\build\bin\Release\MemoChatQml.exe
```

### 2.6 服务停止

```powershell
# 停止后端服务
scripts\windows\stop_test_services.bat

# 停止所有服务
scripts\windows\stop_all_services.bat
```

## 三、Docker Compose 部署

### 3.1 Docker Compose 文件结构

```
deploy/
├── local/
│   └── compose/
│       ├── datastores.yml      # 数据存储服务
│       ├── kafka.yml           # Kafka/Redpanda
│       ├── rabbitmq.yml        # RabbitMQ
│       ├── observability.yml   # 可观测性
│       ├── livekit.yml          # LiveKit
│       └── nginx.yml           # Nginx 负载均衡
└── images/
    ├── Dockerfile              # Nginx 镜像
    └── Dockerfile.gatedrogon   # GateServer 镜像
```

### 3.2 启动数据存储

```powershell
# 启动 PostgreSQL, Redis, MongoDB
docker compose -f .\deploy\local\compose\datastores.yml up -d

# 验证服务
docker ps
```

### 3.3 启动消息队列

```powershell
# 启动 Redpanda (Kafka 兼容)
docker compose -f .\deploy\local\compose\kafka.yml up -d

# 启动 RabbitMQ
docker compose -f .\deploy\local\compose\rabbitmq.yml up -d
```

### 3.4 启动可观测性

```powershell
# 启动 Prometheus, Grafana, Loki, Tempo
docker compose -f .\deploy\local\compose\observability.yml up -d
```

### 3.5 服务容器化

#### 3.5.1 构建服务端镜像

```bash
# 构建 GateServer 镜像
docker build -f deploy/images/Dockerfile.gatedrogon.linux \
  -t memochat/gateserver:latest .

# 构建 ChatServer 镜像
docker build -f deploy/images/Dockerfile.gatedrogon.linux \
  -t memochat/chatserver:latest .
```

#### 3.5.2 Docker Compose 服务编排

```yaml
# docker-compose.yml
version: '3.8'

services:
  gateserver:
    image: memochat/gateserver:latest
    ports:
      - "8080:8080"
    environment:
      - POSTGRES_HOST=postgres
      - REDIS_HOST=redis
      - KAFKA_BROKERS=redpanda:19092
    depends_on:
      - postgres
      - redis
      - redpanda

  statusserver:
    image: memochat/statusserver:latest
    ports:
      - "50052:50052"

  chatserver1:
    image: memochat/chatserver:latest
    environment:
      - NODE_NAME=chatserver1
    ports:
      - "8090:8090"
      - "8190:8190"
      - "50055:50055"

  chatserver2:
    image: memochat/chatserver:latest
    environment:
      - NODE_NAME=chatserver2
    ports:
      - "8091:8091"
      - "8191:8191"
      - "50056:50056"

  varifyserver:
    image: memochat/varifyserver:latest
    ports:
      - "50051:50051"

  postgres:
    image: postgres:16
    environment:
      - POSTGRES_USER=memochat
      - POSTGRES_PASSWORD=123456
      - POSTGRES_DB=memo_pg
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7.2.5
    command: redis-server --requirepass 123456

  redpanda:
    image: redpandadata/redpanda:v24.2.6
    ports:
      - "19092:19092"

  rabbitmq:
    image: rabbitmq:3.12-management-alpine
    ports:
      - "5672:5672"
      - "15672:15672"

volumes:
  postgres_data:
```

## 四、Kubernetes 部署

### 4.1 Kubernetes 资源结构

```
Memo_ops/k8s/
├── base/                    # 基础资源配置
│   ├── namespace.yaml
│   ├── configmap.yaml
│   └── secrets.yaml
├── backend/                 # 后端服务部署
│   ├── gateserver.yaml
│   ├── statusserver.yaml
│   ├── chatserver.yaml
│   └── varifyserver.yaml
├── infra/                   # 基础设施
│   ├── postgres.yaml
│   ├── redis.yaml
│   ├── mongodb.yaml
│   ├── kafka.yaml
│   └── rabbitmq.yaml
├── etcd/                    # etcd 服务发现
│   └── statefulset.yaml
├── overlays/                # 环境覆盖
│   ├── dev/
│   ├── dev-single/
│   ├── staging/
│   └── prod/
```

### 4.2 Helm Chart

```
helm/memochat/
├── Chart.yaml
├── values.yaml              # 默认配置
└── values/
    ├── dev.yaml            # 开发环境
    ├── staging.yaml        # 预发布环境
    └── prod.yaml           # 生产环境
```

### 4.3 部署前准备

#### 4.3.1 创建命名空间

```bash
kubectl create namespace memochat
kubectl create namespace memochat-infra
```

#### 4.3.2 配置 Secrets

```yaml
# base/secrets.yaml
apiVersion: v1
kind: Secret
metadata:
  name: memochat-secrets
  namespace: memochat
type: Opaque
stringData:
  postgres-password: "memochat-prod-password"
  redis-password: "memochat-prod-redis-password"
  chat-auth-secret: "memochat-prod-chat-secret"
```

### 4.4 使用 Kustomize 部署

#### 4.4.1 开发环境部署

```bash
# 使用 dev overlay
kubectl apply -k Memo_ops/k8s/overlays/dev

# 查看部署状态
kubectl get pods -n memochat
```

#### 4.4.2 生产环境部署

```bash
# 使用 prod overlay
kubectl apply -k Memo_ops/k8s/overlays/prod
```

### 4.5 使用 Helm 部署

#### 4.5.1 安装 Chart

```bash
# 添加 Helm 仓库（如果使用远程仓库）
helm repo add memochat https://charts.example.com/memochat

# 安装开发环境
helm install memochat-dev helm/memochat \
  --values helm/memochat/values/dev.yaml \
  --namespace memochat-dev \
  --create-namespace

# 安装生产环境
helm install memochat-prod helm/memochat \
  --values helm/memochat/values/prod.yaml \
  --namespace memochat-prod \
  --create-namespace
```

#### 4.5.2 升级部署

```bash
helm upgrade memochat-prod helm/memochat \
  --values helm/memochat/values/prod.yaml \
  --namespace memochat-prod
```

#### 4.5.3 回滚

```bash
# 查看发布历史
helm history memochat-prod

# 回滚到上一个版本
helm rollback memochat-prod

# 回滚到指定版本
helm rollback memochat-prod 3
```

### 4.6 服务配置

#### 4.6.1 GateServer 配置

```yaml
# backend/gateserver.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: gateserver
  namespace: memochat
spec:
  replicas: 2
  selector:
    matchLabels:
      app: gateserver
  template:
    spec:
      containers:
      - name: gateserver
        image: memochat/gateserver:latest
        ports:
        - containerPort: 8080
        - containerPort: 50054
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "1000m"
        env:
        - name: POSTGRES_HOST
          value: "postgres.memochat-infra.svc.cluster.local"
        - name: REDIS_HOST
          value: "redis.memochat-infra.svc.cluster.local"
```

#### 4.6.2 ChatServer 配置

```yaml
# backend/chatserver.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: chatserver
  namespace: memochat
spec:
  serviceName: chatserver
  replicas: 4
  selector:
    matchLabels:
      app: chatserver
  template:
    spec:
      containers:
      - name: chatserver
        image: memochat/chatserver:latest
        ports:
        - containerPort: 8090  # TCP
          name: tcp
        - containerPort: 8190  # QUIC
          name: quic
        - containerPort: 50055  # gRPC
          name: grpc
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
```

#### 4.6.3 水平自动扩缩容

```yaml
# gateserver.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: gateserver-hpa
  namespace: memochat
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: gateserver
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

### 4.7 Ingress 配置

```yaml
# ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: memochat-ingress
  namespace: memochat
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/proxy-body-size: "100m"
spec:
  ingressClassName: nginx
  rules:
  - host: api.memochat.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: gateserver
            port:
              number: 8080
  tls:
  - hosts:
    - api.memochat.example.com
    secretName: memochat-tls
```


## 五、服务发现与负载均衡

### 5.1 服务间发现

#### 5.1.1 Kubernetes Service

```yaml
# ChatServer Service
apiVersion: v1
kind: Service
metadata:
  name: chatserver
  namespace: memochat
spec:
  clusterIP: None  # Headless Service
  selector:
    app: chatserver
  ports:
  - name: tcp
    port: 8090
    targetPort: 8090
  - name: quic
    port: 8190
    targetPort: 8190
  - name: grpc
    port: 50055
    targetPort: 50055
```

#### 5.1.2 etcd 服务注册

```bash
# ChatServer 启动时注册到 etcd
etcdctl put /memochat/chatserver/1 "{\"host\":\"10.0.0.1\",\"port\":8090}"
etcdctl put /memochat/chatserver/2 "{\"host\":\"10.0.0.2\",\"port\":8091}"

# StatusServer 查询 ChatServer 列表
etcdctl get /memochat/chatserver/
```

### 5.2 客户端负载均衡

GateServer 通过轮询或最少连接策略选择 ChatServer：


### 6.3 告警规则

```yaml
# alert-rules.yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: memochat-alerts
  namespace: memochat
spec:
  groups:
  - name: memochat
    rules:
    - alert: HighErrorRate
      expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.05
      for: 5m
      labels:
        severity: critical
      annotations:
        summary: "High error rate detected"
        description: "Error rate is above 5% for 5 minutes"

    - alert: ServiceDown
      expr: up{job="gateserver"} == 0
      for: 1m
      labels:
        severity: critical
      annotations:
        summary: "GateServer is down"

    - alert: HighMemoryUsage
      expr: container_memory_usage_bytes{job="gateserver"} > 0.8
      for: 5m
      labels:
        severity: warning
      annotations:
        summary: "High memory usage"
```

## 七、灾难恢复

### 7.1 数据备份策略

| 数据类型 | 备份频率 | 保留时间 | 存储位置 |
|---------|----------|----------|----------|
| PostgreSQL | 每日全量 | 30 天 | S3/本地 |
| MongoDB | 每小时增量 | 7 天 | S3/本地 |
| Redis RDB | 每小时 | 24 小时 | 本地 |
| Kafka 消息 | 保留 7 天 | 7 天 | Kafka 集群 |

### 7.2 恢复流程

#### 7.2.1 PostgreSQL 恢复

```bash
# 停止服务
kubectl scale deployment gateserver --replicas=0 -n memochat

# 执行恢复
pg_restore -h postgres.memochat-infra.svc.cluster.local \
  -U memochat -d memo_pg /backups/memo_pg_backup.sql

# 启动服务
kubectl scale deployment gateserver --replicas=2 -n memochat
```

#### 7.2.2 Kubernetes 故障恢复

```bash
# 查看故障 Pod
kubectl get pods -n memochat

# 删除故障 Pod（会自动重建）
kubectl delete pod gateserver-xxxx -n memochat

# 查看 Pod 日志
kubectl logs gateserver-xxxx -n memochat

# 执行 Pod 调试
kubectl exec -it gateserver-xxxx -n memochat -- /bin/bash
```
