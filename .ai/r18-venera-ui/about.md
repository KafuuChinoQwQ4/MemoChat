# R18 Venera UI Adaptation

The R18 section borrows Venera's product structure without coupling MemoChat to Venera's Flutter code. The R18 face uses a MemoChat-style transparent acrylic/glass look and a single leftmost icon rail with four pages: 主页, 本地, 导航, and 数据源. These buttons use PNG resources from `apps/client/desktop/MemoChat-qml/src` and are registered through qrc aliases. The R18 rail also keeps a bottom return-to-chat button so the user can switch back to the normal chat face at any time; this button deliberately matches the conversion button used on the chat sidebar. The old R18-side chat sidebar and the old Canvas-drawn inner navigation rail have been removed, leaving one clear navigation source.

R18 content uses translucent white glass panels, blue-gray hover/selected states, and gray/dark typography for readability over the bright transparent glass background. The normal chat sidebar now uses the same 72px white-glass icon rail style as the R18 sidebar, and both sidebar surfaces preserve press-and-drag window movement through `Window.window.startSystemMove()`. Detail and reader remain drill-in subpages behind the four-page navigation flow. Source management can directly pull a comic-source JavaScript URL such as `https://cdn.jsdelivr.net/gh/venera-app/venera-configs@main/jm.js`; the catalog view remains an optional discovery helper. Downloaded scripts are staged through MemoChat's R18 import route as `venera-js`.

The staged JavaScript source files are not yet executable search/detail/page providers. Real source execution requires a later Venera JS API runtime adapter that implements the `Network`, source parser, comic model, image loading, and account hooks expected by Venera source scripts. R18 remains isolated behind `R18Controller`, `R18ListModel`, and the existing `/api/r18/*` gateway routes.
